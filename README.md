# Flow · Instagram Design System (HTML-кит)

Дизайн-система контента Instagram для студии **Flow** на HTML/CSS. Шаблоны редактируются
текстом, рендерятся в PNG нужного размера для постинга и легко переносятся в Figma вручную.

Полная спецификация решений: `../intdoc_corporate/docs/superpowers/specs/2026-06-18-flow-instagram-design-system-design.md`

## Структура

```
flow_instagram_ds/
├─ css/
│  ├─ tokens.css   ← единый источник правды: цвета, градиенты, шрифты, темы
│  └─ kit.css      ← компоненты и классы (рамки, фото-оверлей, типографика, pill, аватар…)
├─ stories.html    ← Phase 1: 6 сторис 1080×1920 (тёмная + светлая тема)
├─ profile.html    ← Профиль-витрина на телефоне (тёмная / светлая тема)
├─ conversion.html ← Конверсионный слой: «до / после» по рычагам 1–3
├─ render.mjs      ← экспорт всех шаблонов в PNG
└─ out/            ← сюда падают PNG (создаётся при экспорте)
```

## Принципы (кратко)

- **Две роли шрифта.** Playfair Display *italic* — эмоция (заголовки, цитаты). Jost — функция
  (время, цена, CTA, тело). Критичную инфо НЕ ставим серифом по фото — только Jost по скриму.
- **Фото-центрично.** Реальное фото девушек на фоне + единый оверлей (дуотон ~35% + скрим),
  чтобы лента из разных фото не «рассыпалась».
- **Темы.** Тёмная (по умолчанию) и светлая — переключатель в шапке `stories.html` или `?theme=light`.

## Как редактировать

1. Открой `stories.html` в браузере. В шапке — тумблер темы и масштаб превью.
2. Текст меняется прямо в разметке шаблона.
3. **Фото:** на блоке `.photo` задай переменную:
   ```html
   <div class="photo" style="--photo: url('assets/nastya.jpg') center/cover"></div>
   ```
   Сила фирменного дуотона: класс `duo` (35%, реком.), `duo--light` (мягко), `duo--strong` (максимум бренда).
4. **Цвет/шрифт глобально** — правишь `css/tokens.css`, меняется везде.

## Экспорт в PNG

```bash
npm init -y && npm i puppeteer
node render.mjs          # обе темы → out/story-*-dark.png, out/story-*-light.png
node render.mjs dark     # только тёмная
```
Каждый `.frame` экспортируется в натуральную величину (сторис = 1080×1920).

## Перенос в Figma

Когда снимется лимит Figma-плана (или после апгрейда) — токены из `tokens.css` ложатся в
color/text styles, компоненты из `kit.css` — в Figma-компоненты, шаблоны — в фреймы. Кит служит
точным визуальным эталоном.
